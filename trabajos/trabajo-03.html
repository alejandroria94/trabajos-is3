Trabajo en Clase 03

Introducción práctica a Docker y Docker Compose con aplicación Java y MySQL

← Volver al listado

Modalidad: Grupal (2–4 estudiantes)

Duración: 90–120 minutos

Herramientas: IntelliJ IDEA, Docker Desktop, Maven

Entrega: Un (1) documento PDF breve por grupo con capturas y respuestas

1. Objetivo

Comprender cómo Docker permite empaquetar aplicaciones en contenedores y cómo Docker Compose facilita la ejecución coordinada de varios servicios, incluyendo una aplicación Java y una base de datos MySQL.

2. Contexto de Ingeniería de Software

Uno de los problemas más comunes en el desarrollo de software es:

“En mi computador funciona”.

Esto ocurre porque pueden cambiar entre entornos:

  • la versión de Java
  • las dependencias instaladas
  • la configuración del servidor
  • el sistema operativo
  • la disponibilidad de servicios auxiliares como bases de datos

Docker ayuda a empaquetar la aplicación y su entorno de ejecución. Docker Compose ayuda a levantar varios servicios relacionados como una sola solución.

3. Conceptos básicos

Concepto Descripción
Imagen Plantilla con sistema base, runtime, dependencias y aplicación.
Contenedor Instancia en ejecución de una imagen.
Dockerfile Archivo que define cómo construir una imagen.
Puerto Canal que permite acceder a un servicio desde fuera del contenedor.
Volumen Mecanismo para persistir datos fuera del ciclo de vida del contenedor.
Docker Compose Herramienta para definir y levantar varios servicios con un solo archivo YAML.

4. Requisitos previos

  • Docker Desktop instalado y en ejecución.
  • IntelliJ IDEA instalado.
  • Proyecto Java abierto en IntelliJ.
  • Maven disponible desde IntelliJ o desde la terminal integrada.

5. Estructura esperada del proyecto

El proyecto debe abrirse en IntelliJ y la raíz del proyecto debe contener los archivos principales. La ubicación sugerida en Windows es: 

C:\LABS\DemoAcademicoApplication

La estructura esperada es:

DemoAcademicoApplication
│
├── src
├── target
├── pom.xml
├── Dockerfile
└── docker-compose.yml

Importante: El archivo Dockerfile y el archivo docker-compose.yml deben quedar en la raíz del proyecto, al mismo nivel de pom.xml.

6. Verificar instalación de Docker

En la terminal integrada de IntelliJ ejecutar:

docker --version
docker info

7. Primer contenedor de prueba

Ejecutar:

docker run hello-world

Este comando verifica que Docker puede descargar una imagen, crear un contenedor y ejecutarlo correctamente.

8. Contenedor web de ejemplo

Ejecutar:

docker run -d -p 8088:80 --name mi-nginx nginx

Luego abrir en el navegador:

http://localhost:8088

Este ejemplo permite observar un servicio web ejecutándose dentro de un contenedor.

9. Dos contenedores para comprender puertos

Ejecutar:

docker run -d -p 8088:80 --name nginx1 nginx
docker run -d -p 8089:80 --name nginx2 nginx

Probar en el navegador:

http://localhost:8088
http://localhost:8089

Observe que ambos contenedores usan internamente el puerto 80, pero el host necesita puertos distintos.

10. Comandos básicos de Docker

Comando Descripción
docker ps Lista contenedores en ejecución.
docker ps -a Lista todos los contenedores, incluso detenidos.
docker images Lista imágenes disponibles localmente.
docker logs nombre Muestra la salida y los logs de un contenedor.
docker stop nombre Detiene un contenedor.
docker start nombre Inicia un contenedor detenido.
docker rm nombre Elimina un contenedor.
docker rmi imagen Elimina una imagen.
docker build -t nombre . Construye una imagen a partir de un Dockerfile.

11. Compilar la aplicación Java

En la terminal integrada de IntelliJ ejecutar:

mvn clean package

Este comando debe generar un archivo .jar dentro de la carpeta: 

target/

Verificación: revise en IntelliJ que dentro de target exista un archivo .jar. Ese archivo será copiado al contenedor.

12. Crear el Dockerfile

Crear en la raíz del proyecto el archivo Dockerfile con el siguiente contenido:

FROM eclipse-temurin:21-jdk
RUN mkdir -p /opt/demo-academico
ENV TZ="America/Bogota"
ENV WORKSPACE /opt/demo-academico
WORKDIR ${WORKSPACE}
COPY target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java","-Xmx1024m","-jar","app.jar"]

13. Explicación del Dockerfile

Instrucción Explicación
FROM eclipse-temurin:21-jdk Usa una imagen base con Java 21 instalado.
RUN mkdir -p /opt/demo-academico Crea el directorio interno de trabajo.
ENV TZ="America/Bogota" Configura la zona horaria del contenedor.
ENV WORKSPACE /opt/demo-academico Define una variable con la ruta de trabajo.
WORKDIR ${WORKSPACE} Establece el directorio de trabajo.
COPY target/*.jar app.jar Copia el JAR compilado al contenedor con el nombre app.jar.
EXPOSE 8080 Declara el puerto usado por la aplicación.
ENTRYPOINT [...] Define el comando de inicio del contenedor.

14. Construir la imagen

Ejecutar:

docker build -t demoacademico .

Luego verificar:

docker images

15. Ejecutar el contenedor de la aplicación

Ejecutar:

docker run -d -p 8080:8080 --name demoacademico demoacademico

Verificar:

docker ps
docker logs demoacademico

Probar en navegador o Postman:

http://localhost:8080

16. Introducción a Docker Compose

En aplicaciones reales, no siempre basta con un solo contenedor. Muchas veces se necesita ejecutar varios servicios al mismo tiempo, por ejemplo:

  • una aplicación backend
  • una base de datos
  • un servidor web

Docker Compose permite describir todos esos servicios en un solo archivo llamado docker-compose.yml y ejecutarlos con un único comando.

Idea principal: con Docker Compose, la arquitectura básica del sistema queda declarada en un archivo YAML.

17. Crear el archivo docker-compose.yml

En la raíz del proyecto crear el archivo docker-compose.yml con el siguiente contenido:

services:

  app:
    image: demoacademico
    container_name: demoacademico-app
    ports:
      - "8080:8080"
    depends_on:
      - db

  db:
    image: mysql:8.0
    container_name: demoacademico-db
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: demoacademico
      MYSQL_USER: demo
      MYSQL_PASSWORD: demo123
    ports:
      - "3307:3306"
    volumes:
      - mysql_data:/var/lib/mysql

volumes:
  mysql_data:

18. Explicación del docker-compose.yml

Elemento Explicación
services Define los servicios o contenedores que componen la solución.
app Servicio de la aplicación Java construido previamente.
db Servicio de base de datos MySQL.
depends_on Indica que la aplicación depende del servicio de base de datos.
environment Define variables de entorno para inicializar MySQL.
ports Publica puertos del contenedor hacia el host.
volumes Permite persistir los datos de MySQL.

19. Sobre el mapeo de puertos en MySQL

En el servicio de base de datos se usa:

"3307:3306"

Esto significa:

  • 3307: puerto del host
  • 3306: puerto interno del contenedor MySQL

Se usa 3307 en el host para evitar conflictos si el equipo ya tiene una instalación local de MySQL escuchando en el puerto 3306.

20. Ejecutar Docker Compose

Desde la terminal ejecutar:

docker compose up -d

Verificar servicios:

docker compose ps

Ver logs:

docker compose logs

21. Verificar ejecución de los servicios

Comprobar que la aplicación responde en:

http://localhost:8080

Comprobar que el contenedor MySQL está activo con:

docker ps

También se puede revisar solo el log de MySQL:

docker logs demoacademico-db

22. Detener todos los servicios

docker compose down

Este comando detiene y elimina los contenedores creados por Docker Compose. El volumen persistente de MySQL puede mantenerse para conservar los datos.

23. Diferencia entre Docker y Docker Compose

Docker Docker Compose
Trabaja normalmente con contenedores individuales. Gestiona varios servicios relacionados.
Usa comandos como docker run. Usa comandos como docker compose up.
La configuración suele quedar distribuida en comandos. La configuración queda declarada en un archivo YAML.
