Trabajo en Clase 03

Contenerización de una aplicación con Docker

← Volver al listado

Modalidad: Grupal (2-4 estudiantes)

Duración: 90 minutos

Entrega: Capturas de pantalla + breve reflexión

1. Objetivo

Comprender el uso de Docker como herramienta para empaquetar aplicaciones, garantizando portabilidad, reproducibilidad y consistencia entre entornos de desarrollo, pruebas y producción.

2. Contexto de Ingeniería de Software

Uno de los problemas más comunes en el desarrollo de software es la inconsistencia entre entornos de ejecución.

Ejemplo típico:

  • En el computador del desarrollador funciona correctamente
  • En el servidor de pruebas aparecen errores de configuración
  • En producción existen dependencias diferentes

Docker permite resolver este problema empaquetando la aplicación y su entorno de ejecución dentro de un contenedor.

3. Conceptos fundamentales

Concepto Descripción
Imagen Plantilla que contiene el sistema base, dependencias y aplicación.
Contenedor Instancia en ejecución de una imagen.
Dockerfile Archivo que define cómo construir una imagen.
Puerto Permite acceder a la aplicación desde fuera del contenedor.
Volumen Mecanismo para persistir datos generados por la aplicación.

4. Paso 1 — Verificar instalación de Docker

Ejecutar en la terminal:

docker --version

Luego verificar que Docker esté ejecutándose:

docker info

5. Paso 2 — Compilar la aplicación

Ubíquese en el directorio del proyecto y ejecute:

mvn clean package

Este comando genera el archivo compilado en:

target/demoacademico.jar

6. Paso 3 — Crear el Dockerfile

Crear un archivo llamado Dockerfile con el siguiente contenido:

FROM eclipse-temurin:21-jdk

RUN mkdir -p /opt/demo-academico

ENV TZ="America/Bogota"
ENV WORKSPACE /opt/demo-academico

WORKDIR ${WORKSPACE}

COPY target/*.jar app.jar

EXPOSE 8080

ENTRYPOINT ["java","-Xmx1024m","-jar","app.jar"]

7. Paso 4 — Construir la imagen

Ejecutar:

docker build -t demoacademico .

Luego verificar las imágenes disponibles:

docker images

8. Paso 5 — Ejecutar el contenedor

Ejecutar el siguiente comando:

docker run -d -p 8080:8080 --name demoacademico demoacademico

Explicación de parámetros:

Parámetro Descripción
-d Ejecuta el contenedor en segundo plano.
-p 8080:8080 Mapea el puerto del contenedor al host.
--name Nombre del contenedor.
demoacademico Imagen utilizada.

9. Paso 6 — Verificar ejecución

Ejecutar:

docker ps

Luego abrir en el navegador:

http://localhost:8080

10. Paso 7 — Visualizar logs

docker logs demoacademico

Este comando permite observar el proceso de arranque del servicio.

11. Discusión

Responder en grupo:
  1. ¿Cuál es la diferencia entre una imagen y un contenedor?
  2. ¿Qué problema de ingeniería de software soluciona Docker?
  3. ¿Por qué Docker es importante en arquitecturas basadas en microservicios?
  4. ¿Qué ventajas ofrece Docker frente a ejecutar la aplicación directamente en el sistema operativo?
