Trabajo en Clase 03

Introducción práctica a Docker y Docker Compose con aplicación Java y MySQL

← Volver al listado

Modalidad: Grupal (2–4 estudiantes)

Duración: 90–120 minutos

Herramientas: IntelliJ IDEA, Docker Desktop, Maven

Entrega: Un (1) documento PDF breve por grupo con capturas y respuestas

1. Objetivo

Comprender cómo Docker permite empaquetar aplicaciones y sus dependencias dentro de contenedores, y cómo Docker Compose facilita la ejecución coordinada de múltiples servicios como una aplicación Java y una base de datos MySQL.

2. Contexto de Ingeniería de Software

Uno de los problemas más comunes en el desarrollo de software es la frase:

“En mi computador funciona”.

Esto ocurre porque entre desarrollo, pruebas y producción pueden cambiar:

  • la versión de Java
  • las librerías instaladas
  • la configuración del sistema
  • el sistema operativo
  • los servicios auxiliares como bases de datos

Docker ayuda a resolver este problema empaquetando la aplicación junto con su entorno de ejecución. Docker Compose ayuda a coordinar varios servicios relacionados dentro de una misma solución.

3. Conceptos básicos

Concepto Descripción
Imagen Plantilla que contiene el sistema base, librerías, dependencias y aplicación necesarias para ejecutar un software.
Contenedor Instancia en ejecución de una imagen.
Dockerfile Archivo que define paso a paso cómo construir una imagen Docker personalizada.
Puerto Canal que permite acceder a un servicio desde fuera del contenedor.
Volumen Mecanismo de almacenamiento persistente que permite conservar datos incluso si el contenedor se elimina.
Docker Compose Herramienta para definir y ejecutar múltiples contenedores relacionados usando un archivo YAML.

4. Requisitos previos

  • Docker Desktop instalado y ejecutándose.
  • IntelliJ IDEA instalado.
  • Proyecto Java abierto en IntelliJ.
  • Maven funcional desde IntelliJ o desde la terminal integrada.

5. Estructura esperada del proyecto

La ruta sugerida en Windows es:

C:\LABS\DemoAcademicoApplication

La estructura del proyecto debe verse así:

DemoAcademicoApplication
│
├── src
├── target
├── pom.xml
├── Dockerfile
└── docker-compose.yml

Importante: El archivo Dockerfile y el archivo docker-compose.yml deben quedar en la raíz del proyecto, al mismo nivel de pom.xml.

6. Verificar instalación de Docker

Desde la terminal integrada de IntelliJ ejecutar:

docker --version
docker info

Estos comandos permiten comprobar que Docker está instalado y que el motor de Docker se encuentra activo.

7. Primer contenedor de prueba

Ejecutar:

docker run hello-world

Este comando descarga una imagen de prueba, crea un contenedor y muestra un mensaje confirmando que Docker funciona correctamente.

Aprendizaje esperado:

  • Docker puede descargar imágenes.
  • Docker puede crear contenedores a partir de imágenes.
  • No todos los contenedores quedan ejecutándose permanentemente.

8. Ejecutar un servidor web en un contenedor

Ejecutar:

docker run -d -p 8088:80 --name mi-nginx nginx

Luego abrir en el navegador:

http://localhost:8088

Esto permite observar un servicio web ejecutándose dentro de un contenedor Docker.

9. Ejecutar dos contenedores para comprender puertos

Ejecutar:

docker run -d -p 8088:80 --name nginx1 nginx
docker run -d -p 8089:80 --name nginx2 nginx

Probar en navegador:

http://localhost:8088
http://localhost:8089

Ambos contenedores usan internamente el puerto 80, pero el host necesita puertos distintos.

10. Explicación detallada del mapeo de puertos

En Docker, el mapeo de puertos sigue esta estructura:

puertoHost:puertoContenedor

Ejemplo:

8088:80

Significa:

  • 8088: puerto del computador (host)
  • 80: puerto interno del contenedor

Entonces:

http://localhost:8088  --->  contenedor:80

Esto se hace porque el contenedor está aislado y su puerto interno no es visible automáticamente desde el host. Docker crea una redirección entre el puerto del computador y el puerto del contenedor.

Ejemplo Interpretación
8080:8080 El puerto 8080 del host redirige al puerto 8080 del contenedor.
8088:80 El puerto 8088 del host redirige al puerto 80 del contenedor.
3307:3306 El puerto 3307 del host redirige al puerto 3306 del contenedor MySQL.

Importante: Si dos contenedores quieren publicar un mismo puerto del host, se producirá un conflicto. Por eso, al usar varios contenedores similares, se asignan puertos distintos en el host.

11. Comandos básicos de Docker

Comando Descripción
docker ps Lista contenedores en ejecución.
docker ps -a Lista todos los contenedores, incluso los detenidos.
docker images Lista las imágenes disponibles localmente.
docker logs nombre Muestra la salida y los logs de un contenedor.
docker stop nombre Detiene un contenedor.
docker start nombre Inicia un contenedor detenido.
docker restart nombre Reinicia un contenedor.
docker rm nombre Elimina un contenedor.
docker rmi imagen Elimina una imagen.
docker volume ls Lista los volúmenes existentes en Docker.

12. Compilar la aplicación Java

Desde la terminal integrada de IntelliJ ejecutar:

mvn clean package

Este comando compila la aplicación y genera un archivo .jar en la carpeta: 

target/

Verificación: revise en IntelliJ que dentro de target exista un archivo .jar. Ese será el archivo que Docker copiará al contenedor.

13. Crear el Dockerfile

Crear en la raíz del proyecto el archivo Dockerfile con el siguiente contenido:

FROM eclipse-temurin:21-jdk
RUN mkdir -p /opt/demo-academico
ENV TZ="America/Bogota"
ENV WORKSPACE /opt/demo-academico
WORKDIR ${WORKSPACE}
COPY target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java","-Xmx1024m","-jar","app.jar"]

14. Explicación línea por línea del Dockerfile

Instrucción Explicación
FROM eclipse-temurin:21-jdk Define la imagen base. En este caso se usa una imagen que ya trae Java 21 instalado.
RUN mkdir -p /opt/demo-academico Crea un directorio dentro del contenedor donde se almacenará la aplicación.
ENV TZ="America/Bogota" Define la zona horaria del contenedor.
ENV WORKSPACE /opt/demo-academico Crea una variable de entorno con la ruta de trabajo.
WORKDIR ${WORKSPACE} Indica que a partir de ese momento Docker trabajará dentro de esa carpeta.
COPY target/*.jar app.jar Copia el archivo JAR compilado al contenedor y lo renombra como app.jar.
EXPOSE 8080 Declara que la aplicación utilizará el puerto 8080 dentro del contenedor.
ENTRYPOINT ["java","-Xmx1024m","-jar","app.jar"] Define el comando que se ejecutará cuando el contenedor inicie.

15. Construir la imagen de la aplicación

Ejecutar:

docker build -t demoacademico .

Luego verificar:

docker images

La imagen llamada demoacademico debería aparecer en el listado.

16. Ejecutar el contenedor de la aplicación

Ejecutar:

docker run -d -p 8080:8080 --name demoacademico demoacademico

Verificar:

docker ps
docker logs demoacademico

Probar en navegador o Postman:

http://localhost:8080

17. Introducción a Docker Compose

En aplicaciones reales, muchas veces no basta con un solo contenedor. Una solución completa puede requerir:

  • una aplicación backend
  • una base de datos
  • un servidor web
  • otros servicios auxiliares

Docker Compose permite describir todos esos servicios en un solo archivo llamado docker-compose.yml y ejecutarlos juntos.

Idea principal: Docker Compose convierte la arquitectura de despliegue en una configuración declarativa dentro de un archivo YAML.

18. Crear el archivo docker-compose.yml

Crear en la raíz del proyecto el archivo docker-compose.yml con el siguiente contenido:

services:

  app:
    image: demoacademico
    container_name: demoacademico-app
    ports:
      - "8080:8080"
    depends_on:
      - db

  db:
    image: mysql:8.0
    container_name: demoacademico-db
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: demoacademico
      MYSQL_USER: demo
      MYSQL_PASSWORD: demo123
    ports:
      - "3307:3306"
    volumes:
      - mysql_data:/var/lib/mysql

volumes:
  mysql_data:

19. Explicación detallada del docker-compose.yml

Elemento Explicación
services Define los servicios o contenedores que forman parte de la solución.
app Nombre lógico del servicio de la aplicación Java.
image: demoacademico Indica que el servicio app utilizará la imagen previamente construida.
container_name: demoacademico-app Asigna un nombre explícito al contenedor de la aplicación.
ports: "8080:8080" Publica el puerto 8080 del contenedor en el puerto 8080 del host.
depends_on: - db Indica que el servicio app depende del servicio db.
db Nombre lógico del servicio de base de datos.
image: mysql:8.0 Indica que el servicio usará la imagen oficial de MySQL 8.0.
container_name: demoacademico-db Asigna un nombre explícito al contenedor MySQL.
environment Define variables de entorno que inicializan MySQL al crear el contenedor.
MYSQL_ROOT_PASSWORD Contraseña del usuario root de MySQL.
MYSQL_DATABASE Base de datos que MySQL creará automáticamente.
MYSQL_USER Usuario adicional que se creará automáticamente.
MYSQL_PASSWORD Contraseña del usuario adicional.
ports: "3307:3306" Publica el puerto 3306 del contenedor MySQL en el puerto 3307 del host.
volumes: - mysql_data:/var/lib/mysql Monta un volumen persistente en la carpeta interna donde MySQL guarda sus datos.
volumes Declara el volumen global llamado mysql_data.

20. Explicación detallada del servicio MySQL

MySQL guarda su información en la ruta interna: 

/var/lib/mysql

Cuando se define: 

mysql_data:/var/lib/mysql

se está indicando que los datos de MySQL se almacenarán en un volumen llamado mysql_data.

Esto permite que la información persista aunque el contenedor sea eliminado y recreado.

Muy importante: Las bases de datos no deberían guardar información solo dentro del contenedor, porque al eliminar el contenedor también se perderían los datos. Por eso se usan volúmenes.

21. Explicación de imágenes, contenedores y volúmenes

Elemento Qué es Ejemplo en esta actividad Para qué se utiliza
Imagen Plantilla que contiene el sistema base, dependencias y software. 
image: demoacademico
image: mysql:8.0
Permite construir contenedores idénticos en cualquier computador.
Contenedor Instancia en ejecución de una imagen. 
container_name: demoacademico-app
container_name: demoacademico-db
Es el entorno donde realmente se ejecutan la aplicación y la base de datos.
Volumen Almacenamiento persistente fuera del ciclo de vida del contenedor. 
mysql_data:/var/lib/mysql
Permite conservar la información de MySQL aunque el contenedor se elimine o se vuelva a crear.

Relación entre estos elementos:

  • La imagen es la plantilla.
  • El contenedor es la ejecución real de esa plantilla.
  • El volumen conserva la información generada por el contenedor.

22. Diagrama conceptual de la solución

docker-compose.yml
        │
        ▼
 ┌──────────────────────────────┐
 │       Docker Compose         │
 └──────────────────────────────┘
        │
        ├───────────────┬────────────────
        ▼               ▼
 ┌──────────────┐  ┌──────────────┐
 │ app          │  │ db           │
 │ demoacademico│  │ mysql:8.0    │
 └──────────────┘  └──────────────┘
        │               │
        │               ▼
        │        ┌──────────────┐
        │        │ mysql_data   │
        │        │ volumen      │
        │        └──────────────┘
        │
        ▼
 http://localhost:8080

En esta arquitectura:

  • app representa la aplicación Java.
  • db representa la base de datos MySQL.
  • mysql_data representa el volumen persistente de la base de datos.

23. Ejecutar Docker Compose

Ejecutar:

docker compose up -d

Verificar servicios:

docker compose ps
docker ps
docker logs demoacademico-app
docker logs demoacademico-db

Verificar volúmenes:

docker volume ls

24. Detener los servicios

docker compose down

Este comando detiene y elimina los contenedores creados por Compose. El volumen puede mantenerse para conservar los datos.

25. Diferencia entre Docker y Docker Compose

Docker Docker Compose
Se usa normalmente para ejecutar contenedores individuales. Se usa para ejecutar y coordinar múltiples servicios relacionados.
Utiliza comandos como docker run. Utiliza comandos como docker compose up.
La configuración suele quedar distribuida en varios comandos. La configuración queda centralizada en un archivo YAML.
